# Decision 002 — 先用 gemma4:e4b，之後再試 gemma4:26b

## 背景

已確定模型世代為 Gemma 4，需要在 RTX 2070 SUPER 8GB VRAM 的限制下選擇先部署的型號。

## Input

| 項目 | 內容 |
|---|---|
| GPU | RTX 2070 SUPER |
| VRAM | 8GB |
| RAM | 62GB |
| 使用者偏好 | 先裝穩定且容易跑起來的版本，再加試更大模型 |

## 比較

| 模型 | 推估 Q4 VRAM | 適配情況 |
|---|---|---|
| `gemma4:e2b` | 約 3.2GB | 可完整進 GPU，但能力較保守 |
| `gemma4:e4b` | 約 5.0GB | 可完整進 GPU，速度與能力平衡最佳 |
| `gemma4:26b` | 約 15.6GB | 需 CPU offload，可做第二階段測試 |
| `gemma4:31b` | 約 17.4GB | 不適合目前機器 |

## DA 表

| 欄位 | 內容 |
|---|---|
| 技術/決策的採用前提 | GPU 僅 8GB VRAM，但主機 RAM 很充足 |
| 採用目的(效果) | 先以低風險方式啟動可用模型，再保留升級測試空間 |
| 結果影響內容 | 第一個落地型號是 `gemma4:e4b`；大型模型改為後續驗證項 |
| 結果影響範圍 | 模型拉取、推論速度、後續測試路線 |
| 採用成本 | 低 |

## 結論

第一階段先部署 **`gemma4:e4b`**；等基礎流程穩定後，再加試 **`gemma4:26b`**。
