# Decision 005 — 所有報告檔案改用流水號命名

## 背景

雖然前一輪已把內容拆成多個獨立檔案，但檔名仍無法直接反映它們發生的先後順序。使用者要求所有檔案都能從檔名看出順序，因此需要建立一致的流水號系統。

## Input

| 項目 | 內容 |
|---|---|
| 使用者要求 | 所有檔案都能透過檔名知道其發生的順序 |
| 現況問題 | 原檔名只有類型與主題，沒有全域時間序列 |

## DA 表

| 欄位 | 內容 |
|---|---|
| 技術/決策的採用前提 | 檔案將持續增加，且需要長期維持可追溯性 |
| 採用目的(效果) | 讓閱讀者不開檔也能從檔名理解事件順序 |
| 結果影響內容 | 所有 `./session/` 內報告改為帶全域流水號的檔名 |
| 結果影響範圍 | 既有報告命名、後續新增報告、交接閱讀方式 |
| 採用成本 | 低 |

## 流水號規格

| 項目 | 規格 |
|---|---|
| 格式 | `{4位流水號}-{type}-{topic}.md` |
| 範例 | `0009-decision-install-ollama.md` |
| 分配方式 | 每新增一個檔案就取下一號 |
| 是否允許重編 | 否 |
| 是否允許插號 | 否 |

## 結論

從現在起，`./session/` 內所有報告與決策檔都採用 **四位數全域流水號命名**，並以此作為唯一的順序依據。
