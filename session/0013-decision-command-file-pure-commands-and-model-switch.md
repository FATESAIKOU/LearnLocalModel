# Decision 006 — command.md 改成純指令串並支援模型切換

## 背景

使用者明確要求 `command.md` 應該是一連串可直接複製執行的指令，而不是帶有 Input / Output / Process 結構的說明文件。同時，使用者希望這份指令檔能支援從 `gemma4:e4b` 切換到 `gemma4:26b`。

## Input

| 項目 | 內容 |
|---|---|
| 使用者要求 1 | `command.md` 要是純指令串 |
| 使用者要求 2 | 需要能切換到 Gemma 4 26B |
| 已確認模型 tag | `gemma4:26b` |

## DA 表

| 欄位 | 內容 |
|---|---|
| 技術/決策的採用前提 | 使用者主要透過複製 `command.md` 內容來執行操作 |
| 採用目的(效果) | 降低複製成本，讓同一份檔案同時覆蓋 E4B 與 26B 操作 |
| 結果影響內容 | `command.md` 不再承載教學敘述，只保留可執行指令與少量註解 |
| 結果影響範圍 | 後續所有交付給使用者執行的命令 |
| 採用成本 | 低 |

## 結論

`command.md` 已改為純指令串格式，並提供：

1. `gemma4:e4b` 驗證指令
2. `gemma4:26b` 下載指令
3. `gemma4:e4b` / `gemma4:26b` 切換測試方式

後續只要持續維持這個模式即可。
