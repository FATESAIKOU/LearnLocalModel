# Decision 004 — 每個決策各自獨立成檔，統一放在 ./session/

## 背景

最初曾把多個決策集中寫在同一份報告內，但使用者明確要求改為「每次做的一次決定，對應一個檔案」。

## Input

| 項目 | 內容 |
|---|---|
| 使用者要求 1 | 每次問答拿到答案後，都要產出報告並補上之前的決策 |
| 使用者要求 2 | 報告全部寫到專案內的 `./session/` 下 |
| 使用者要求 3 | 每次做的一次決定，都要對應一個獨立檔案 |

## DA 表

| 欄位 | 內容 |
|---|---|
| 技術/決策的採用前提 | 需要讓未參與討論的人也能閱讀與接手 |
| 採用目的(效果) | 讓每個決策具備獨立上下文、理由與結論，便於追蹤 |
| 結果影響內容 | 不再把全部決策塞進單一檔案；改為索引檔 + 多個決策檔 |
| 結果影響範圍 | 後續所有報告與交接方式 |
| 採用成本 | 低 |

## 結論

從現在起：

1. 每個新決策都建立一個新的 `.md` 檔案。
2. 所有檔案都放在專案內的 `./session/`。
3. 索引檔僅保留索引與目前狀態，不再承載全部決策內容。
