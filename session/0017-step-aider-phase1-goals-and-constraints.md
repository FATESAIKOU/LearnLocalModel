# Step 008 — Aider 階段一：目標與約束確認

## 背景

使用者要求新任務改為建置 Aider 結對編程環境，並且必須嚴格採用「一問一答」方式分四個階段進行。每個階段結束後都要停下來，等待使用者確認，不能自行推進。

## Input

| 項目 | 內容 |
|---|---|
| 任務主題 | 建置 Aider 結對編程環境 |
| 互動規則 | 一次只做一個階段，階段結束必須等待確認 |
| 架構要求 | 雙引擎熱切換：本地 Ollama + 雲端 GitHub Copilot |
| 設定原則 | 盡量使用原生 Aider 設定，不綁定特定語言框架 |
| 本地模型 | `gemma4:e4b` |

## 階段一結論

| 項目 | 結論 |
|---|---|
| Aider 配置風格 | 採用原生 Aider 設定 |
| 專案語言策略 | 語言無關，不額外引入語言專屬設定 |
| 本地模型策略 | 先以 `gemma4:e4b` 作為預設模型 |
| 雲端模型策略 | 保留 GitHub Copilot 訂閱模型做熱切換 |

## 新增規則

使用者補充要求：**每個階段至少都要生成一個 `session/` 檔案**，因為最後需要彙整成此專案的配置與使用說明。
